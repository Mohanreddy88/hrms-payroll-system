================================================================================
                            WHY DOCKER WORKS
================================================================================

Nixpacks tries to auto-detect your project, but sometimes gets confused with:
- Multiple folders (HrmsApi, hrms-ui)
- .NET SDK version detection
- Build vs Runtime separation

Docker gives explicit instructions:
✅ Use .NET 8 SDK to build
✅ Copy HrmsApi folder
✅ Build in Release mode
✅ Run migrations on startup
✅ Start application

No guessing = Reliable builds!

================================================================================
                          WHAT'S IN THE DOCKERFILE
================================================================================

Stage 1 (Build):
- Uses .NET 8 SDK
- Restores NuGet packages
- Builds HrmsApi project
- Publishes to /app/HrmsApi/out

Stage 2 (Runtime):
- Uses .NET 8 SDK (includes EF tools for migrations)
- Copies published app
- Installs dotnet-ef tool
- Runs entrypoint.sh on startup
  → Waits for database
  → Runs migrations
  → Starts application
