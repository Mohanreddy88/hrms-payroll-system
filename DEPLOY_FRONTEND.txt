================================================================================
              STEP 7: UPDATE BACKEND CORS (IMPORTANT!)
================================================================================

Your backend needs to allow requests from the frontend domain.

On your computer:

1. Open: C:\Users\HP\source\repos\walnut\HrmsApi\Program.cs

2. Find the CORS section (around line 42-49)

3. Update it to include your frontend URL:

   builder.Services.AddCors(options =>
   {
       options.AddPolicy("AllowAngular", policy =>
           policy.WithOrigins(
               "http://localhost:4200",
               "https://YOUR-FRONTEND-URL.up.railway.app"  // ← ADD THIS
           )
           .AllowAnyHeader()
           .AllowAnyMethod()
           .AllowCredentials());
   });

4. Save, commit, push:
   git add HrmsApi/Program.cs
   git commit -m "Add production frontend to CORS"
   git push

5. Railway backend will auto-redeploy (~3 minutes)

================================================================================
                        TROUBLESHOOTING
================================================================================

Frontend shows blank page:
  → Open browser console (F12), check for errors
  → Verify API URL in environment.prod.ts is correct

CORS errors:
  → Make sure frontend URL is in backend CORS policy
  → Backend must be redeployed after CORS update

Login doesn't work:
  → Check backend logs for errors
  → Verify DATABASE_URL is set and PostgreSQL is running
  → Test backend directly: https://backend-url/swagger

Build fails:
  → Check "Deployments" logs for specific error
  → Verify npm install completes successfully
  → Make sure Root Directory = hrms-ui
